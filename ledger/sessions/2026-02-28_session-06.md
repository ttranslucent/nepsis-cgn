## What We Got Done

- New backend service module:
  - `src/nepsis_cgn/api/service.py`
  - Supports:
    - `create_session(...)`
    - `list_sessions()`
    - `get_session(session_id)`
    - `step_session(session_id, ...)`
    - `reframe_session(session_id, ...)`
    - `get_packets(session_id)`
  - Family-specific sign parsing:
    - `puzzle`, `clinical`, `safety`
  - Step responses include:
    - manifold/governor trace
    - governance block (when enabled)
    - iteration packet (when emitted)
    - updated session summary
- New HTTP server:
  - `src/nepsis_cgn/api/server.py`
  - Endpoints:
    - `GET /v1/health`
    - `POST /v1/sessions`
    - `GET /v1/sessions`
    - `GET /v1/sessions/{id}`
    - `POST /v1/sessions/{id}/step`
    - `POST /v1/sessions/{id}/reframe`
    - `GET /v1/sessions/{id}/packets`
  - Includes CORS headers for frontend integration.
- New package init:
  - `src/nepsis_cgn/api/__init__.py`
- New runtime builder shared by CLI and API:
  - `src/nepsis_cgn/core/runtime.py`
- CLI refactor to use shared runtime builder:
  - `src/nepsis_cgn/cli/main.py`
- New script entrypoint:
  - `pyproject.toml` adds `nepsiscgn-api = "nepsis_cgn.api.server:entrypoint"`
- README updates:

- Added tests:
