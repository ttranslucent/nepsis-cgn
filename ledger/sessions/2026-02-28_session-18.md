# Session Ledger - 2026-02-28 (Session 18)

## What We Worked On

- Backend production hardening in three waves: P0, P1, P2.

## What We Got Done

### P0 (Critical Security and Runtime Guardrails)

- Fail-closed auth posture in HTTP server (`src/nepsis_cgn/api/server.py`):
  - Startup now fails when auth is required but `NEPSIS_API_TOKEN` is missing.
  - Anonymous mode requires explicit `NEPSIS_API_ALLOW_ANON=true`.
- CORS allowlist enforcement:
  - Cross-origin requests are denied unless `Origin` is in `NEPSIS_API_ALLOWED_ORIGINS`.
- Request safety limits:
  - Added body size cap via `NEPSIS_API_MAX_REQUEST_BODY_BYTES`.
  - Added read timeout via `NEPSIS_API_REQUEST_TIMEOUT_SECONDS`.
- Error sanitization:
  - Internal exceptions now return generic `500` with `request_id`; internals are logged server-side.
- Manifest path allowlisting:
  - `manifest_path` validated against approved roots (`NEPSIS_API_ALLOWED_MANIFEST_ROOTS` + default manifests directory).
- Added production ASGI runtime path:
  - New `src/nepsis_cgn/api/asgi.py` and script `nepsiscgn-api-asgi`.
  - Added optional dependencies in `pyproject.toml` (`fastapi`, `uvicorn`, `cryptography`).

### P1 (Durability, Throughput, Control)

- Transactional persistence with SQLite default:
  - Default API store path now `ledger/sessions/engine_api_sessions.db`.
  - `EngineApiService` supports SQLite-backed persistence (WAL + transactional writes).
  - JSON fallback remains for `.json` stores.
- Corruption recovery:
  - Corrupt JSON/SQLite stores are moved aside with `.corrupt.<timestamp>` suffix and service recovers cleanly.
- Pagination and caps:
  - `list_sessions(limit, offset)` and `get_packets(limit, offset)` with max-page-size guard (`NEPSIS_API_MAX_PAGE_SIZE`).
- Rate limiting:
  - Added server rate limiting with configurable window and request max:
    - `NEPSIS_API_RATE_LIMIT_WINDOW_SECONDS`
    - `NEPSIS_API_RATE_LIMIT_MAX_REQUESTS`
- Calibration governance control:
  - Enforced calibration version allowlist via `NEPSIS_API_ALLOWED_CALIBRATION_VERSIONS`.

### P2 (Observability, Governance, Discoverability)

- Structured audit logging + request IDs:
  - HTTP and ASGI paths now emit structured JSON response logs.
  - `X-Request-ID` propagated/generated and returned.
- Retention policy support:
  - Optional automatic retention purge at service init via `NEPSIS_API_RETENTION_SECONDS`.
- OpenAPI discoverability:
  - Added `GET /v1/openapi.json` from generated route metadata.
  - Added Next proxy endpoint `nepsis-web/src/app/api/engine/openapi/route.ts`.

### Frontend/Proxy Alignment

- Next engine routes now forward querystrings where needed:
  - Sessions list (`limit`, `offset`)
  - Packets list (`limit`, `offset`)
  - Session TTL purge (`max_age_seconds`, `dry_run`)
