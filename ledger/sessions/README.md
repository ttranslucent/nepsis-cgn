# Session Ledger Protocol

Purpose: end each Nepsis work session with a concise, durable progress record.

Each session note should include:

1. `What We Worked On`
2. `What We Got Done`
3. `What We Need To Do Next`
4. `Overall Progress`

Filename format:

- `YYYY-MM-DD_session-XX.md`

Notes:

- Keep entries short and specific.
- Reference major files changed.
- Record test status (passed/failed/not run).
