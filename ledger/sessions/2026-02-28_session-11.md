## What We Got Done

- Frontend typed client for Engine API:
  - Added `nepsis-web/src/lib/engineClient.ts`.
  - Includes typed wrappers for:
    - health/routes
    - create/list/get/delete session
    - step/reframe
    - packets
  - Includes unified `EngineClientError` and JSON response handling.
- Frontend hook/state layer:
  - Added `nepsis-web/src/lib/useEngineSession.ts`.
  - Manages state for:
    - `sessions`, `activeSession`, `packets`, `lastStep`
    - `routes`, `healthy`, `loading`, `error`
  - Exposes actions:
    - `refreshHealth`, `refreshRoutes`, `refreshSessions`
    - `createSession`, `loadSession`, `step`, `reframe`, `refreshPackets`, `deleteSession`
- Backend session cleanup endpoint:
  - Added service method `delete_session(...)` in `src/nepsis_cgn/api/service.py`.
  - Added HTTP route `DELETE /v1/sessions/{session_id}` in `src/nepsis_cgn/api/server.py`.
  - Added route manifest entry + CORS method support for `DELETE`.
- Web proxy support:
  - Added `DELETE` handler in `nepsis-web/src/app/api/engine/sessions/[sessionId]/route.ts`.
- Tests:
  - Added service test for delete behavior: `tests/test_engine_api_service.py`.
  - Added server route-manifest assertion for delete route: `tests/test_engine_api_server.py`.
- Docs:
  - Updated root `README.md` endpoint list to include `GET /v1/sessions` and `DELETE /v1/sessions/{session_id}`.
  - Updated `nepsis-web/README.md` API proxy list and added references to new frontend client/hook files.
