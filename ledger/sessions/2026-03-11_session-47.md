## What We Worked On

- Website connectivity review with focus on `/engine` session auth, backend connection clarity, and the broken passwordless login UX.

## What We Got Done

- Replaced the brittle passwordless login code path:
  - Removed in-memory route-module code storage that only worked when both auth routes shared process memory.
  - Added signed login challenge cookies so request/verify works across route invocations.
  - Added optional email delivery support via `RESEND_API_KEY` + `NEPSIS_AUTH_FROM_EMAIL`.
  - Added local preview fallback so dev environments now show the one-time code directly instead of silently relying on terminal logs.
- Added auth/session plumbing:
  - New `GET /api/auth/session` route for client auth state.
  - New `POST /api/auth/logout` route.
  - Header now shows signed-in user and logout control.
- Improved `/engine` connection clarity:
  - Added visible cards for backend reachability, session access, and model-key status.
  - Added explicit sign-in-required banner when engine session controls are locked by auth.
  - Reworded the OpenAI-key banner so it no longer implies the key is required for core engine gating.
- Fixed `/engine` SSR/client hydration mismatch caused by timestamped starter messages.
- Cleaned up lint issues on `/` and `/settings` by avoiding synchronous state writes directly inside effects.
